# Функционал кол-центра по вашим документам

Источники: заметки и конфиги в `D:\BM\billing\Инструкция по CSP\` (IPCC-Documentation, DataStation `task.txt`, CSP, `tns for IVR.txt`, `vpn.txt`, `Решение отчета  1728.txt`, `Deleting lock file for convert files.txt`, корневые операционные заметки). Учётные данные из шпаргалок в документ не включены.

## Голосовой контакт-центр (IPCC)

- **Маршрутизация и обслуживание вызовов (ядро IPCC)**: контур **ICD** (`/home/icd/icddir/`) и **CTI**; после сбоев или перезагрузки отдельно поднимают **`icdcomm`** (обмен данными контакт-центра с остальной платформой).
- **Очереди и нагрузка**: из дампов **`QueueInfo*.unl`** метрики очередей и вызовов загружаются в Oracle UIDB процедурой **`p_PrestatAgentCallInfo`**; для мониторинга используется **`p_monitor_PrestatAgentCallInfo`** по файлам `QueueInfo*.old` (схема: `QueueInfo*.unl` переименовывается в `.old`, затем обрабатывается мониторинговой задачей).
- **Работа оператора по вызову**: **`oplist*.unl`** → **`p_PrestatAgentOprInfo`** (операции и состояния по идентификатору вызова `ICDCALLID`).
- **Качество и дисциплина операторов**:
  - **`AgentListen*.unl`** → **`p_prestatagentlisten`** (прослушивание и контроль прослушивания);
  - **`AgentAssess*.unl`** → **`p_PrestatAgentAssessEx`** (оценки качества);
  - **`AgentStat*.unl`** → **`p_PrestatAgentWorkInfo`** (агрегированная статистика работы).
- **Автоматические сценарии (IVR / автоинформатор)**: **`CallInfo*.unl`** → **`p_PrestatAutoInfo`** (сводные показатели по автообработке).
- **Взаимопомощь / супервизия**: **`helpInfo*.unl`** → **`p_PrestatAgentInterHelpInfo`**.
- **Дополнительная статистика**: **`InnoStat*.unl`** → **`spInnoCallStatInfoIns`** (отдельный набор показателей в вашей сборке).

## Детализированный учёт вызовов (CTI CDR)

- Поток **`ctiserverbill`** (`*.unl`, фиксированная запись, поля включают **`ICDCALLID`** и временные метки) → **`p_BillLogPrestat`**: **CDR / журнал CTI** для отчётности, SLA и разборов инцидентов.

## Интеграция с биллингом и хранилищем данных

- **Huawei DataStation PROXY** по расписанию **FTP-скачивает** `.unl` с основного и резервного IPCC-узла (задачи `ccs bill download` и `ctis bill download` в `d:\BM\billing\Инструкция по CSP\IPCC-Documentation\DataStation V100R001C04B030 client\task.txt`), парсит бинарные строки и **пакетно записывает в Oracle UIDB** (`uidb_oracle`, сервис `ora11g`) через перечисленные процедуры.
- **Практический смысл**: единое **хранилище метрик и CDR** для отчётов, контроля очередей, нагрузки, QA и расследований.

## CSP (портал и публикация отчётов)

- **Веб CSP** (в заметках: `http://172.16.3.54/csp`): типичный **операторский / супервизорский портал** Huawei CSP (детальный перечень экранов в ваших `.txt` не зафиксирован; полнота — по продуктовой документации).
- **Oracle CSPPUB / CTI**, пользователь **`ICDPUB`**: таблицы **`T_B_DLG_*`**, **`T_C_DLG_*`**, **`T_CHF_*`**; для отчётов по переведённым вызовам — процедура **`P_B_DLG_CCT_TRANFERRED`** (см. `d:\BM\billing\Инструкция по CSP\IPCC-Documentation\Решение отчета  1728.txt`).

## IVR и интеграция с Oracle

- Алиас **`CSPPUB`** в `tnsnames`: узлы `172.16.3.48` и `172.16.3.49`, порт `1521`, `LOAD_BALANCE`, `SERVICE_NAME=ora11g` (`d:\BM\billing\Инструкция по CSP\tns for IVR.txt`). IVR и интегрированные сценарии **работают с данными в Oracle** (справочники, сессии, результаты меню).

## CRM и короткие номера

- БД **`crmdb`**, таблица **`inf_sub_relation`**: сопоставление **короткий номер ↔ MSISDN** (`d:\BM\billing\Инструкция по CSP\IPCC-Documentation\vpn.txt`) — поддержка **сквозной идентификации абонента** между CRM и кол-центром.

## Параллельные каналы (смежный контур эксплуатации)

- **USSD / SMS (SMPP)**: сервис-ключи вида `*366` / `#366`, параметры кодировки (в т.ч. UCS2) — **самообслуживание и маркетинг** параллельно голосовому IPCC (корневые операционные заметки).
- **Голосовые промпты и ошибки**: учёт и аналитика через **`bm_voice`**, таблица **`error_call`**, плюс отдельный **HTTP API промптов** — **качество медиа и персонализация** поверх IPCC.

## Обслуживание CSP

- Удаление **`csp.lock`** при зависании конвертации или выгрузки (`d:\BM\billing\Инструкция по CSP\IPCC-Documentation\Deleting lock file for convert files.txt`) — операционная процедура для **фоновых задач CSP**.
